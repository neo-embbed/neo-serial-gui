# Core Transport Layer 设计概要

## transport.h — 抽象传输层基类

- `TransportState` 枚举：`Closed / Open / Error`
- `Direction` 枚举：`Rx / Tx / Sys`
- `Message` 结构体：id、时间戳、方向、内容
- `Transport` 抽象类：
  - 纯虚接口：`open()` / `close()` / `write(uint8_t*, size_t)`
  - 便捷重载：`write(vector)` / `write(string)`
  - 回调注册：`onStateChanged()` / `onMessage()`
  - 公开的 `emitMessage()` — 调用方可用来发 Tx/Sys 消息

## uart_transport.h — UART 声明

- `Parity` / `StopBits` 枚举
- `UartConfig` 结构体：port、baudrate、databits、parity、stopbits、超时
- `PortInfo` 结构体：device + description
- `UartTransport` 类：继承 Transport，包含读写线程、写入队列、平台句柄

## uart_transport.cpp — 完整实现

- **open()** — Win32 用 `CreateFileA` + `SetCommState` + `SetCommTimeouts`；POSIX 用 `open` + `tcsetattr`。配置完成后启动读/写线程
- **close()** — 关闭句柄（自动中断阻塞的 ReadFile/read），join 线程，清空队列
- **write()** — 线程安全地将数据推入写队列（有界，最大 1024 帧）
- **readerLoop()** — 独立线程持续读取，通过 `emitMessage(Rx, ...)` 回调上层
- **writerLoop()** — 独立线程消费写队列，condition_variable 等待通知
- **listPorts()** — Win32 读注册表 `SERIALCOMM`；POSIX 扫描 `/dev/tty*`

## 典型使用模式（后期 Qt 集成时）

```cpp
neo::UartConfig cfg;
cfg.port     = "COM3";
cfg.baudrate = 115200;

auto uart = std::make_unique<neo::UartTransport>(cfg);

uart->onMessage([](const neo::Message& msg) {
    // 连接到 Qt signal，更新 UI
});

uart->onStateChanged([](neo::TransportState s, const std::string& detail) {
    // 状态变化通知
});

uart->open();

// 发送数据
std::string text = "hello\n";
uart->emitMessage(neo::Direction::Tx, text);  // 记录到消息流
uart->write(text);                             // 实际发送

uart->close();
```

## 编译备注

Windows 下需链接 `advapi32`（注册表枚举端口用）。
